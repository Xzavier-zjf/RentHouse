# RentHouse 课程增强版开发文档

## 1. 系统架构

项目采用前后端分离和轻量微服务结构：

- `vue-frontend`：Vue 3 + Vite + Element Plus 前端。
- `gateway`：Spring Cloud Gateway，统一代理 `/api/user/**` 和 `/api/house/**`。
- `user-service`：用户注册、登录、个人资料、房东申请和管理员用户审核。
- `house-service`：房源、收藏、预约、租赁申请和管理员统计。
- `db-init`：MySQL 初始化脚本。

增强版继续使用现有技术栈，后端持久层沿用 `JdbcTemplate`。

## 2. 数据库设计

### 2.1 user 表

用于保存用户基础信息和房东申请状态。

关键字段：

- `id`：用户 ID。
- `username`：用户名，唯一。
- `password`：BCrypt 加密密码。
- `email`：邮箱。
- `role`：`USER`、`LANDLORD`、`ADMIN`。
- `landlord_apply_status`：`NOT_APPLIED`、`PENDING`、`APPROVED`、`REJECTED`。
- `created_at`、`updated_at`：创建和更新时间。

### 2.2 house 表

用于保存房源信息、审核状态和出租状态。

关键字段：

- `title`、`price`、`location`：基础展示和搜索字段。
- `description`、`layout`、`area`、`floor`、`orientation`：详情和筛选字段。
- `image_url`：图片地址。
- `contact_name`、`contact_phone`：房源联系人。
- `user_id`：发布房东 ID。
- `status`：`PENDING`、`APPROVED`、`REJECTED`、`OFFLINE`。
- `rent_status`：`AVAILABLE`、`RESERVED`、`RENTED`。
- `reason`：管理员拒绝原因。

### 2.3 favorite 表

用于保存用户收藏关系。

- `user_id`：收藏用户。
- `house_id`：被收藏房源。
- 使用唯一索引 `uk_favorite_user_house` 防止重复收藏。

### 2.4 appointment 表

用于保存预约看房记录。

- `user_id`：预约用户。
- `house_id`：预约房源。
- `landlord_id`：房源所属房东。
- `appointment_time`：看房时间。
- `contact_name`、`contact_phone`、`message`：预约联系信息。
- `status`：`PENDING`、`CONFIRMED`、`CANCELLED`、`FINISHED`。
- `reply`：房东回复。

### 2.5 rental_application 表

用于保存租赁申请。

- `user_id`：申请用户。
- `house_id`：申请房源。
- `landlord_id`：房源所属房东。
- `applicant_name`、`applicant_phone`：申请人信息。
- `move_in_date`、`lease_months`：入住日期和租期。
- `message`：申请说明。
- `status`：`PENDING`、`APPROVED`、`REJECTED`、`CANCELLED`。
- `reply`：房东回复。

## 3. 后端接口设计

### 3.1 用户接口

- `POST /api/user/register`：用户注册。
- `POST /api/user/login`：用户登录。
- `GET /api/user/profile`：获取个人资料。
- `PUT /api/user/profile`：更新个人资料。
- `POST /api/user/apply-landlord`：申请成为房东。
- `GET /api/user/admin/users`：管理员查看用户列表。
- `POST /api/user/admin/approve-landlord?userId={id}`：管理员批准房东申请。

### 3.2 房源接口

- `GET /api/house/search`：搜索已审核房源，支持 `location`、`minPrice`、`maxPrice`、`layout`、`minArea`、`maxArea`、`status`。
- `GET /api/house/{id}`：房源详情。
- `POST /api/house/upload`：房东发布房源。
- `GET /api/house/my-houses`：房东查看自己的房源。
- `PUT /api/house/{id}`：房东编辑自己的房源，编辑后重新待审核。
- `POST /api/house/{id}/offline`：房东下架自己的房源。
- `GET /api/house/admin/houses`：管理员查看房源，可按 `status` 筛选。
- `POST /api/house/admin/approve-house`：管理员审核房源。

### 3.3 收藏接口

- `POST /api/house/{id}/favorite`：收藏房源。
- `DELETE /api/house/{id}/favorite`：取消收藏。
- `GET /api/house/{id}/favorite`：查看是否已收藏。
- `GET /api/house/favorites`：查看我的收藏。

### 3.4 预约接口

- `POST /api/house/appointments`：提交预约看房。
- `GET /api/house/appointments/my`：用户查看自己的预约。
- `POST /api/house/appointments/{id}/cancel`：用户取消预约。
- `GET /api/house/appointments/landlord`：房东查看预约。
- `POST /api/house/appointments/{id}/status`：房东处理预约。

### 3.5 租赁申请接口

- `POST /api/house/rental-applications`：提交租赁申请。
- `GET /api/house/rental-applications/my`：用户查看自己的申请。
- `POST /api/house/rental-applications/{id}/cancel`：用户取消申请。
- `GET /api/house/rental-applications/landlord`：房东查看申请。
- `POST /api/house/rental-applications/{id}/status`：房东处理申请。
- `GET /api/house/admin/rental-applications`：管理员查看所有申请。

### 3.6 统计接口

- `GET /api/house/admin/statistics`：管理员查看用户、房东、房源、预约和申请统计。

## 4. 前端页面设计

### 4.1 首页

首页展示平台名称、登录注册入口和已审核房源列表。未登录用户可以按位置搜索房源，但不能收藏、预约和申请。

### 4.2 用户中心

包含以下标签页：

- 房源搜索：筛选房源、查看详情、收藏、预约、提交申请。
- 我的收藏：查看和取消收藏。
- 我的预约：查看预约状态和取消待处理预约。
- 我的申请：查看租赁申请状态和取消待处理申请。
- 申请房东：提交房东申请。
- 个人信息：维护邮箱。

### 4.3 房东中心

包含以下标签页：

- 上传房源：填写完整房源信息并提交审核。
- 我的房源：查看审核状态、编辑、下架。
- 预约管理：确认、拒绝或完成看房预约。
- 租赁申请：通过或拒绝租赁申请。
- 个人信息：维护邮箱。

### 4.4 管理员中心

包含以下标签页：

- 数据概览：查看用户、房源、预约和申请统计。
- 用户审核：批准房东申请。
- 房源审核：按状态筛选房源，批准或拒绝房源。
- 所有房源：查看平台房源。
- 个人信息：维护邮箱。

## 5. 鉴权与权限规则

- 所有登录用户通过 `Authorization: Bearer <token>` 传递 JWT。
- `USER` 可以收藏、预约和提交租赁申请。
- `LANDLORD` 可以发布房源、管理自己的房源、处理自己房源下的预约和申请。
- `ADMIN` 可以审核用户、审核房源和查看统计。
- 后端通过 JWT 中的 `role` 和 `userId` 进行权限判断。

## 6. 运行与验证

### 6.1 数据库初始化

新库执行：

执行：

```sql
source db-init/init.sql;
```

旧版数据库升级执行：

```sql
source db-init/upgrade-enhanced.sql;
```

升级脚本使用 `IF NOT EXISTS`，可在增强版结构已存在时重复执行，用于补齐缺失字段、表和默认管理员账号。

默认管理员：

- 用户名：`admin`
- 密码：`admin123`

### 6.2 后端启动

分别启动三个服务：

```bash
cd user-service
mvn spring-boot:run

cd house-service
mvn spring-boot:run

cd gateway
mvn spring-boot:run
```

### 6.3 前端启动

```bash
cd vue-frontend
npm install
npm run dev
```

### 6.4 验证流程

推荐按以下顺序验证：

1. 注册普通用户并登录。
2. 普通用户申请成为房东。
3. 管理员登录并批准房东申请。
4. 房东发布房源。
5. 管理员审核通过房源。
6. 普通用户搜索房源、收藏、预约、提交租赁申请。
7. 房东处理预约和租赁申请。
8. 管理员查看统计概览。
